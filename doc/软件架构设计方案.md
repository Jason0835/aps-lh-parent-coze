## 1. 系统概述

### 1.1 系统背景

硫化排程系统（APS-LH）是轮胎制造行业的核心生产计划系统，负责将月度生产计划分解为可执行的每日、每班次机台生产任务。系统需要综合考虑多种约束条件，包括：

- **资源约束**：硫化机数量、模具可用性、人员配置
- **时间约束**：换模能力、设备保养、首检时间
- **工艺约束**：结构切换、胶囊使用次数、胎胚库存
- **业务约束**：优先级排序、发货要求、供应链调整

### 1.2 核心业务场景

根据需求文档，系统主要处理以下核心场景：

1. **T/T+1日硫化排程调整** - 根据夜班完成情况动态调整当日计划
2. **T+2日硫化排程生成** - 基于月度计划生成未来排程
3. **设备计划停机处理** - 保养、维修、精度校验
4. **模具清洗计划** - 干冰清洗、喷砂清洗
5. **模具交替计划** - 换模、换活字块
6. **开停产管控** - 特殊时期的产能调整
7. **试制量试排产** - 新产品试制计划
8. **异常处理** - 设备故障、胎胚问题

### 1.3 技术栈

- **开发语言**：Java 8+
- **框架**：Spring Boot 2.x
- **持久层**：MyBatis Plus
- **数据库**：MySQL
- **构建工具**：Maven
- **项目管理**：Maven多模块

## 2. 架构设计原则

### 2.1 核心设计原则

本架构设计遵循以下核心原则：

1. **开闭原则（OCP）**：对扩展开放，对修改关闭
2. **单一职责原则（SRP）**：每个类只负责一个功能领域
3. **依赖倒置原则（DIP）**：依赖抽象而非具体实现
4. **接口隔离原则（ISP）**：客户端不应依赖不需要的接口
5. **里氏替换原则（LSP）**：子类可以替换父类而不影响程序正确性

### 2.2 架构分层

采用经典的分层架构模式：

```
┌─────────────────────────────────────────────────────────┐
│                     表现层 (Controller)                   │
│  - 接收请求、参数校验、返回响应                              │
├─────────────────────────────────────────────────────────┤
│                     应用层 (Service)                      │
│  - 编排领域逻辑、事务控制、跨领域协调                        │
├─────────────────────────────────────────────────────────┤
│                     领域层 (Domain/Engine)                │
│  - 排程算法、约束计算、规则引擎                              │
├─────────────────────────────────────────────────────────┤
│                     数据层 (Repository/Mapper)            │
│  - 数据访问、持久化、缓存                                   │
└─────────────────────────────────────────────────────────┘
```

## 4. 设计模式应用

### 4.1 策略模式（Strategy Pattern）

#### 应用场景

策略模式用于处理不同的排程算法和约束计算逻辑：

1. **不同场景下的排程策略**：续作排程 vs 新增排程
2. **不同优先级的排序策略**：高优先级、中优先级、周期性排产
3. **不同设备的产能计算策略**：单模、双模、不同硫化时间
4. **不同的约束校验策略**：换模约束、首检约束、清洗约束

#### 设计实现

```java
/**
 * 排程策略接口
 * 定义排程算法的标准接口
 */
public interface ScheduleStrategy {
    
    /**
     * 执行排程
     * @param context 排程上下文
     * @return 排程结果
     */
    ScheduleResult execute(ScheduleContext context);
    
    /**
     * 获取策略类型
     * @return 策略类型枚举
     */
    StrategyType getStrategyType();
    
    /**
     * 是否支持该场景
     * @param context 上下文
     * @return true-支持, false-不支持

    boolean supports(ScheduleContext context);
}

/**
 * 新增规格排程策略
 * 处理新上机SKU的排程
 */
@Component
public class NewSkuScheduleStrategy implements ScheduleStrategy {
    
    @Autowired
    private PrioritySorter prioritySorter;
    
    @Autowired
    private MachineMatcher machineMatcher;
    
    @Autowired
    private ChangeMouldBalancer changeMouldBalancer;
    
    @Override
    public ScheduleResult execute(ScheduleContext context) {
        // 1. 优先级排序
        List<SkuPlan> sortedSkus = prioritySorter.sort(context.getNewSkus());
        
        // 2. 筛选可用机台
        List<ScheduleTask> tasks = new ArrayList<>();
        for (SkuPlan sku : sortedSkus) {
            List<Machine> candidates = machineMatcher.findAvailableMachines(sku, context);
            
            // 3. 换模次数均衡校验
            if (!changeMouldBalancer.checkAndAllocate(sku, candidates, context)) {
                markUnscheduled(sku, UnscheduledReason.CHANGE_MOULD_LIMIT);
                continue;
            }
            
            // 4. 首检均衡分配
            FirstCheckAllocation allocation = allocateFirstCheck(sku, context);
            
            // 5. 计算开产时间并分配计划量
            ScheduleTask task = calculateAndAllocate(sku, candidates, allocation, context);
            tasks.add(task);
        }
        
        return buildResult(context, tasks);
    }
    
    @Override
    public StrategyType getStrategyType() {
        return StrategyType.NEW_SKU;
    }
    
    @Override
    public boolean supports(ScheduleContext context) {
        return context.hasNewSku();
    }
}

/**
 * 策略上下文
 * 负责策略的选择和执行
 */
@Component
public class ScheduleStrategyContext {
    
    @Autowired
    private List<ScheduleStrategy> strategies;
    
    /**
     * 执行排程
     */
    public ScheduleResult schedule(ScheduleContext context) {
        ScheduleResult result = new ScheduleResult();
        
        for (ScheduleStrategy strategy : strategies) {
            if (strategy.supports(context)) {
                ScheduleResult partialResult = strategy.execute(context);
                result.merge(partialResult);
            }
        }
        
        return result;
    }
    
    /**
     * 获取指定类型的策略
     */
    public ScheduleStrategy getStrategy(StrategyType type) {
        return strategies.stream()
            .filter(s -> s.getStrategyType() == type)
            .findFirst()
            .orElseThrow(() -> new BusinessException("未找到策略: " + type));
    }
}
```

### 4.2 工厂模式（Factory Pattern）

#### 应用场景

工厂模式用于对象的创建和管理：

1. **排程上下文创建**：根据排程日期创建对应的上下文对象
2. **约束校验器创建**：根据约束类型创建对应的校验器
3. **处理器创建**：根据处理类型创建对应的处理器
4. **数据加载器创建**：根据不同数据源创建加载器

#### 设计实现

```java
/**
 * 排程上下文工厂
 * 负责创建排程上下文对象
 */
@Component
public class ScheduleContextFactory {
    
    @Autowired
    private MonthPlanLoader monthPlanLoader;
    
    @Autowired
    private MachineInfoLoader machineInfoLoader;
    
    @Autowired
    private WorkCalendarLoader workCalendarLoader;
    
    @Autowired
    private DevicePlanShutLoader devicePlanShutLoader;
    
    @Autowired
    private CleaningPlanLoader cleaningPlanLoader;
    
    /**
     * 创建排程上下文
     * @param scheduleDate 排程日期
     * @param factoryCode 工厂编码
     * @return 排程上下文
     */
    public ScheduleContext createContext(Date scheduleDate, String factoryCode) {
        ScheduleContext context = new ScheduleContext();
        context.setScheduleDate(scheduleDate);
        context.setFactoryCode(factoryCode);
        
        // 加载基础数据
        loadBaseData(context);
        
        // 加载排程数据
        loadScheduleData(context);
        
        // 加载约束数据
        loadConstraintData(context);
        
        return context;
    }
    
    private void loadBaseData(ScheduleContext context) {
        // 加载月生产计划
        List<MonthPlan> monthPlans = monthPlanLoader.load(
            context.getFactoryCode(),
            context.getScheduleDate()
        );
        context.setMonthPlans(monthPlans);
        
        // 加载硫化机台信息
        List<MachineInfo> machines = machineInfoLoader.load(
            context.getFactoryCode()
        );
        context.setMachines(machines);
        
        // 加载工作日历
        WorkCalendar calendar = workCalendarLoader.load(
            context.getFactoryCode(),
            context.getScheduleDate()
        );
        context.setWorkCalendar(calendar);
    }
    
    private void loadScheduleData(ScheduleContext context) {
        // 加载前日排程结果（用于续作判断）
        List<LhScheduleResult> prevResults = loadPrevSchedule(context);
        context.setPrevScheduleResults(prevResults);
        
        // 加载MES在机信息
        List<MachineOnlineInfo> onlineInfos = loadOnlineInfo(context);
        context.setOnlineInfos(onlineInfos);
        
        // 计算续作SKU
        List<SkuPlan> continueSkus = calculateContinueSkus(context);
        context.setContinueSkus(continueSkus);
        
        // 计算新增SKU
        List<SkuPlan> newSkus = calculateNewSkus(context);
        context.setNewSkus(newSkus);
    }
    
    private void loadConstraintData(ScheduleContext context) {
        // 加载设备停机计划
        List<DevicePlanShut> shuts = devicePlanShutLoader.load(
            context.getFactoryCode(),
            context.getScheduleDate()
        );
        context.setDevicePlanShuts(shuts);
        
        // 加载模具清洗计划
        List<CleaningPlan> cleaningPlans = cleaningPlanLoader.load(
            context.getFactoryCode(),
            context.getScheduleDate()
        );
        context.setCleaningPlans(cleaningPlans);
    }
}

/**
 * 约束校验器接口
 */
public interface ConstraintValidator {
    
    /**
     * 校验约束
     * @param task 排程任务
     * @param context 上下文
     * @return 校验结果
     */

    /**
     * 获取约束类型

/**
 * 换模能力约束校验器
 */
@Component("changeMouldValidator")
public class ChangeMouldConstraintValidator implements ConstraintValidator {
    
    @Override
    public ValidationResult validate(ScheduleTask task, ScheduleContext context) {
        // 校验换模次数是否超限
        int dailyChangeCount = context.getDailyChangeMouldCount();
        if (dailyChangeCount >= 15) {
            return ValidationResult.fail("换模次数超过每日限制(15台)");
        }
        
        // 校验早班换模次数
        int morningCount = context.getMorningChangeCount();
        if (morningCount >= 8) {
            return ValidationResult.fail("早班换模次数超过限制(8台)");
        }
        
        // 校验中班换模次数
        int afternoonCount = context.getAfternoonChangeCount();
        if (afternoonCount >= 7) {
            return ValidationResult.fail("中班换模次数超过限制(7台)");
        }
        
        // 校验换模时间窗口（20:00-06:00不换模）
        if (isInNoChangeTimeWindow(task.getPlannedStartTime())) {
            return ValidationResult.fail("换模时间不允许在20:00-06:00之间");
        }
        
        return ValidationResult.success();
    }
    
    @Override
    public String getConstraintType() {
        return "CHANGE_MOULD";
    }
    
    @Override
    public int getPriority() {
        return 1; // 高优先级
    }
}

/**
 * 首检约束校验器
 */
@Component("firstCheckValidator")
public class FirstCheckConstraintValidator implements ConstraintValidator {
    
    @Override
    public ValidationResult validate(ScheduleTask task, ScheduleContext context) {
        // 校验首检时间窗口（中班14:00-20:00）
        if (task.isNeedFirstCheck()) {
            Date startTime = task.getPlannedStartTime();
            if (!isInFirstCheckWindow(startTime)) {
                return ValidationResult.fail("首检时间不在允许窗口内(14:00-20:00)");
            }
            
            // 校验首检均衡
            int firstCheckCount = context.getFirstCheckCount(startTime);
            if (firstCheckCount >= 5) {
                return ValidationResult.fail("该班次首检次数超过限制(5台)");
            }
        }
        
        return ValidationResult.success();
    }
    
    @Override
    public String getConstraintType() {
        return "FIRST_CHECK";
    }
    
    @Override
    public int getPriority() {
        return 2;
    }
}

/**
 * 设备停机约束校验器
 */
@Component("deviceShutValidator")
public class DeviceShutConstraintValidator implements ConstraintValidator {

    public ValidationResult validate(ScheduleTask task, ScheduleContext context) {
        String machineCode = task.getMachineCode();
        Date startTime = task.getPlannedStartTime();

                );
            }
        }
        
        return ValidationResult.success();
    }
    
    @Override
    public String getConstraintType() {
        return "DEVICE_SHUT";
    }
    
    @Override
    public int getPriority() {
        return 3;
    }
}
```

### 4.3 模板方法模式（Template Method Pattern）

#### 应用场景

模板方法模式用于定义排程流程的标准骨架，将可变步骤延迟到子类实现：

1. **排程流程模板**：定义排程的标准流程步骤
2. **数据加载模板**：定义数据加载的标准流程
3. **结果校验模板**：定义结果校验的标准流程
4. **异常处理模板**：定义异常处理的标准流程

#### 设计实现

```java
/**
 * 排程流程模板
 * 定义硫化排程的标准流程骨架
 */
public abstract class AbstractScheduleTemplate {
    
    @Autowired
    protected ScheduleContextFactory contextFactory;
    
    @Autowired
    protected ScheduleValidator scheduleValidator;
    
    @Autowired
    protected ScheduleResultSaver resultSaver;
    
    /**
     * 模板方法 - 定义排程流程骨架
     */
    public final ScheduleResult doSchedule(ScheduleRequest request) {
        // 1. 前置校验与数据清理
        if (!preValidate(request)) {
            throw new BusinessException("前置校验失败，无法进行排程");
        }
        
        // 2. 创建排程上下文
        ScheduleContext context = createContext(request);
        
        // 3. 加载并初始化基础数据
        loadBaseData(context);
        
        // 4. 执行排程前校验
        if (!beforeScheduleValidate(context)) {
            throw new BusinessException("排程前校验失败");
        }
        
        // 5. 执行具体排程逻辑（抽象方法）
        ScheduleResult result = doExecute(context);
        
        // 6. 执行排程后校验
        if (!afterScheduleValidate(result, context)) {
            throw new BusinessException("排程结果校验失败");
        }
        
        // 7. 生成模具交替计划
        generateMouldChangePlan(result, context);
        
        // 8. 保存排程结果
        saveResult(result, context);
        
        // 9. 发布事件（钩子方法）
        publishScheduleEvent(result, context);
        
        return result;
    }
    
    /**
     * 前置校验
     * 检查是否可以进行排程
     */
    protected boolean preValidate(ScheduleRequest request) {
        // 检查MES下发状态
        if (isAlreadyIssued(request.getScheduleDate())) {
            log.error("排程日期{}已下发MES，禁止重新排程", request.getScheduleDate());
            return false;
        }
        
        // 检查是否正在排程
        if (isScheduling(request.getScheduleDate())) {
```

### 4.4 责任链模式（Chain of Responsibility Pattern）

#### 应用场景

责任链模式用于处理排程约束的链式校验：

1. **约束校验链**：多个约束校验器依次执行
2. **数据过滤链**：多个数据过滤器依次处理
3. **异常处理链**：多个异常处理器依次处理

#### 设计实现

```java
/**
 * 责任链接口
 */
public interface ScheduleHandler<T, R> {
    
    /**
     * 设置下一个处理器
     */
    void setNext(ScheduleHandler<T, R> next);
    
    /**
     * 处理请求
     */
    R handle(T request);
    
    /**
     * 是否支持处理该请求
     */
    boolean supports(T request);
}

/**
 * 抽象处理器
 */
public abstract class AbstractScheduleHandler<T, R> implements ScheduleHandler<T, R> {
    
    protected ScheduleHandler<T, R> nextHandler;
    
    @Override
    public void setNext(ScheduleHandler<T, R> next) {
        this.nextHandler = next;
    }
    
    /**
     * 调用下一个处理器
     */
    protected R doNext(T request) {
        if (nextHandler != null && nextHandler.supports(request)) {
            return nextHandler.handle(request);
        }
        return null;
    }
}

/**
 * 约束校验责任链
 */
@Component
public class ConstraintValidationChain {
    
    private ScheduleHandler<ScheduleTask, ValidationResult> firstHandler;
    
    @Autowired
    public void buildChain(List<ConstraintValidator> validators) {
        // 按优先级排序
        List<ConstraintValidator> sortedValidators = validators.stream()
            .sorted(Comparator.comparing(ConstraintValidator::getPriority))
            .collect(Collectors.toList());
        
        // 构建责任链
        ScheduleHandler<ScheduleTask, ValidationResult> current = null;
        for (ConstraintValidator validator : sortedValidators) {
            ValidationHandler handler = new ValidationHandler(validator);
            if (firstHandler == null) {
                firstHandler = handler;
                current = handler;
            } else {
                current.setNext(handler);
                current = handler;
            }
        }
    }
    
    /**
     * 执行校验
     */
    public ValidationResult validate(ScheduleTask task, ScheduleContext context) {
        if (firstHandler == null) {
            return ValidationResult.success();
        }
        
        ValidationResult result = firstHandler.handle(task);
        if (!result.isSuccess()) {
            return result;
        }
        
        return ValidationResult.success();
    }
    
    /**
     * 校验处理器
     */
    private class ValidationHandler extends AbstractScheduleHandler<ScheduleTask, ValidationResult> {
        
        private final ConstraintValidator validator;
        private final ScheduleContext context;
        
        public ValidationHandler(ConstraintValidator validator) {
            this.validator = validator;
        }
        
        @Override
        public ValidationResult handle(ScheduleTask task) {
            // 执行当前校验
            ValidationResult result = validator.validate(task, context);
            
            // 如果校验失败，直接返回
            if (!result.isSuccess()) {
                return result;
            }
            
            // 如果成功，继续下一个校验
            return doNext(task);
        }
        
        @Override
        public boolean supports(ScheduleTask task) {
            return validator.supports(task);
        }
    }
}
```

### 4.5 观察者模式（Observer Pattern）

#### 应用场景

观察者模式用于处理排程过程中的事件通知：

1. **排程完成通知**：通知相关系统排程已完成
2. **异常告警通知**：设备故障、欠产等异常情况通知
3. **计划调整通知**：计划变更时通知相关方
4. **模具交替通知**：换模计划生成后通知执行部门

#### 设计实现

```java
/**
 * 观察者接口
 */

    /**
     * 感兴趣的事件类型
     */
    List<EventType> getInterestedEvents();
    
    /**
     * 处理事件
     */

    /**
     * 获取优先级
     */
    int getPriority();

/**
 * 事件发布器
 */
@Component
public class ScheduleEventPublisher {
    
    private final Map<EventType, List<ScheduleObserver>> observerMap = new HashMap<>();
    
    @Autowired
    public void registerObservers(List<ScheduleObserver> observers) {
        for (ScheduleObserver observer : observers) {
            for (EventType eventType : observer.getInterestedEvents()) {
                observerMap.computeIfAbsent(eventType, k -> new ArrayList<>())
                    .add(observer);
            }
        }
        
        // 按优先级排序
        for (List<ScheduleObserver> list : observerMap.values()) {
            list.sort(Comparator.comparing(ScheduleObserver::getPriority));
        }
    }
    
    /**
     * 发布事件
     */
    public void publish(ScheduleEvent event) {
        List<ScheduleObserver> observers = observerMap.get(event.getEventType());
        if (observers != null) {
            for (ScheduleObserver observer : observers) {
                try {
                    observer.onEvent(event);
                } catch (Exception e) {
                    log.error("事件处理失败: observer={}, event={}", 
                        observer.getClass().getName(), event, e);
                }
            }
        }
    }
}
```

### 4.6 装饰器模式（Decorator Pattern）

#### 应用场景

装饰器模式用于动态添加排程计算的功能：

1. **产能计算器装饰**：添加各种因素对产能的影响计算
2. **时间计算器装饰**：添加各种时间因素的计算
3. **约束校验器装饰**：添加额外的校验逻辑

```java
 * 基础产能计算器

@Component
public class BaseCapacityCalculator implements CapacityCalculator {
    
    @Override
    public Capacity calculate(ScheduleTask task, ScheduleContext context) {
        // 基础产能 = (班次时间 - 非生产作业时间) / 硫化时间 * 模数
        int shiftMinutes = getShiftMinutes(task.getShiftCode());
        int nonProdMinutes = getNonProdMinutes(task);
        int vulcanizationSeconds = task.getVulcanizationTime();
        int mouldCount = getMouldCount(task.getMachineCode());
        
        int availableMinutes = shiftMinutes - nonProdMinutes;
        int cyclesPerShift = (availableMinutes * 60) / vulcanizationSeconds;
        int capacity = cyclesPerShift * mouldCount;
        
        return new Capacity(capacity, availableMinutes);
    }
```

### 5.2 优先级排序模块

#### 5.2.1 设计说明

优先级排序是排程的核心环节之一，需要根据多种因素对SKU进行排序。采用策略模式实现不同的排序算法。

```java
/**
 * 优先级排序器接口
 */
public interface PrioritySorter {
    
    /**
     * 排序SKU列表
     */
    List<SkuPlan> sort(List<SkuPlan> skus);
}

/**
 * 综合优先级排序器
 * 按照需求文档中的优先级规则排序
 */
@Component
public class CompositePrioritySorter implements PrioritySorter {
    
    @Override
    public List<SkuPlan> sort(List<SkuPlan> skus) {
        return skus.stream()
            .sorted(this::comparePriority)
            .collect(Collectors.toList());
    }
    
    private int comparePriority(SkuPlan s1, SkuPlan s2) {
        // 1. 有发货要求的优先（周程滚动调整有锁定上机日期）
        int deliveryCompare = compareDeliveryLockDate(s1, s2);
        if (deliveryCompare != 0) return deliveryCompare;
        
        // 2. 延误上机的优先（延误越久越优先上机）
        int delayCompare = compareDelayDays(s1, s2);
        if (delayCompare != 0) return delayCompare;
        
        // 3. 未来5天结构要收尾的，该结构下的SKU收尾日越晚的优先上机
        int structureCompare = compareStructureFinish(s1, s2);
        if (structureCompare != 0) return structureCompare;
        
        // 4. 供应链优先级（高优先级->周期排产->中优先级->搭配排产）
        return compareSupplyChainPriority(s1, s2);
    }
    
    private int compareDeliveryLockDate(SkuPlan s1, SkuPlan s2) {
        // 有锁定日期的优先
        boolean s1HasLock = s1.getLockMachineDate() != null;
        boolean s2HasLock = s2.getLockMachineDate() != null;
        
        if (s1HasLock && !s2HasLock) return -1;
        if (!s1HasLock && s2HasLock) return 1;
        
        // 都有锁定时，锁定日期早的优先
        if (s1HasLock && s2HasLock) {
            return s1.getLockMachineDate().compareTo(s2.getLockMachineDate());
        }
        
        return 0;
    }
    
    private int compareDelayDays(SkuPlan s1, SkuPlan s2) {
        // 延误越久越优先
        return Integer.compare(s2.getDelayDays(), s1.getDelayDays());
    }
    
    private int compareStructureFinish(SkuPlan s1, SkuPlan s2) {
        // 未来5天结构要收尾的优先
        boolean s1WillFinish = s1.isStructureWillFinishIn5Days();
        boolean s2WillFinish = s2.isStructureWillFinishIn5Days();
        
        if (s1WillFinish && !s2WillFinish) return -1;
        if (!s1WillFinish && s2WillFinish) return 1;
        
        // 都要收尾的，收尾日越晚越优先
        if (s1WillFinish && s2WillFinish) {
            return s2.getStructureFinishDate().compareTo(s1.getStructureFinishDate());
        }
        
        return 0;
    }
    
    private int compareSupplyChainPriority(SkuPlan s1, SkuPlan s2) {
        return Integer.compare(
            s1.getSupplyChainPriority().getPriority(),
            s2.getSupplyChainPriority().getPriority()
        );
    }
}
```

```java
/**

@Component
public class DefaultMachineMatcher implements MachineMatcher {
    
    @Autowired
    private List<MachineFilter> machineFilters;
    
    @Autowired
    private MachineScorer machineScorer;
    
    @Override
    public List<MachineScore> findAvailableMachines(SkuPlan sku, ScheduleContext context) {

        // 2. 依次应用过滤器

        for (MachineFilter filter : machineFilters) {

        // 3. 计算机台得分
        return availableMachines.stream()
            .map(machine -> machineScorer.score(machine, sku, context))
            .sorted(Comparator.comparing(MachineScore::getScore).reversed())
            .collect(Collectors.toList());
    }

        if (CollectionUtils.isEmpty(candidates)) {
            return MachineMatchResult.fail("无可用机台");
        }
        
        // 选择得分最高的机台
        MachineScore best = candidates.get(0);
        return MachineMatchResult.success(best.getMachine(), best.getScore());
    }

@Component
public class MachineScorer {
    
    /**
     * 计算机台得分
     * 评分规则：
     * 1. 收尾时间+-20分钟内的优先
     * 2. 相同规格优先
     * 3. 相同英寸优先
     * 4. 相近英寸优先
     * 5. 胶囊共用性好的优先
     * 6. 胎胚共用多的优先
     */
    public MachineScore score(MachineInfo machine, SkuPlan sku, ScheduleContext context) {
        double score = 0.0;
        List<String> scoreDetails = new ArrayList<>();
        
        // 1. 收尾时间匹配度（最高权重）
        double timeScore = calculateTimeScore(machine, sku, context);
        score += timeScore * 0.3;
        scoreDetails.add("收尾时间得分: " + timeScore);
        
        // 2. 规格匹配度
        double specScore = calculateSpecScore(machine, sku);
        score += specScore * 0.2;
        scoreDetails.add("规格匹配得分: " + specScore);
        
        // 3. 英寸匹配度
        double inchScore = calculateInchScore(machine, sku);
        score += inchScore * 0.2;
        scoreDetails.add("英寸匹配得分: " + inchScore);
        
        // 4. 胶囊使用次数（越少越好）
        double capsuleScore = calculateCapsuleScore(machine);
        score += capsuleScore * 0.15;
        scoreDetails.add("胶囊得分: " + capsuleScore);
        
        // 5. 胎胚共用度
        double embryoScore = calculateEmbryoScore(machine, sku, context);
        score += embryoScore * 0.15;
        scoreDetails.add("胎胚共用得分: " + embryoScore);
        
        return new MachineScore(machine, score, scoreDetails);
    }
    
    private double calculateTimeScore(MachineInfo machine, SkuPlan sku, ScheduleContext context) {
        // 获取机台当前任务的预计收尾时间
        Date finishTime = getMachineFinishTime(machine, context);
        if (finishTime == null) {
            return 100; // 空机台，最高得分
        }
        
        // 计算时间差（分钟）
        long diffMinutes = Math.abs(finishTime.getTime() - sku.getRequiredStartTime().getTime()) / (60 * 1000);
        
        // +-20分钟内得高分
        if (diffMinutes <= 20) {
            return 100;
        } else if (diffMinutes <= 60) {
            return 80;
        } else if (diffMinutes <= 120) {
            return 60;
        } else {
            return 40;
        }
    }
    
    private double calculateSpecScore(MachineInfo machine, SkuPlan sku) {
        // 相同规格得满分
        if (machine.getCurrentSpecCode() != null && 
            machine.getCurrentSpecCode().equals(sku.getSpecCode())) {
            return 100;
        }
        return 50;
    }
    
    private double calculateInchScore(MachineInfo machine, SkuPlan sku) {
        // 相同英寸得满分
        if (machine.getCurrentInch() != null && 
            machine.getCurrentInch().equals(sku.getInch())) {
            return 100;
        }
        // 相近英寸得中等分
        if (isInchSimilar(machine.getCurrentInch(), sku.getInch())) {

        if (useCount < 500) return 80;
```

### 5.4 约束校验模块

#### 5.4.1 设计说明

约束校验模块负责确保排程结果满足各种业务约束。采用责任链模式组合多个校验器。

```java
/**
 * 约束校验引擎
 */
@Component
public class ConstraintValidationEngine {
    
    @Autowired
    private ConstraintValidationChain validationChain;
    
    @Autowired
    private List<GlobalConstraintValidator> globalValidators;
    
    /**
     * 校验单个任务
     */
    public ValidationResult validateTask(ScheduleTask task, ScheduleContext context) {
        return validationChain.validate(task, context);
    }
    
    /**
     * 校验整个排程结果
     */
    public ValidationResult validateResult(ScheduleResult result, ScheduleContext context) {
        // 1. 校验所有任务
        for (ScheduleTask task : result.getTasks()) {
            ValidationResult taskResult = validateTask(task, context);
            if (!taskResult.isSuccess()) {
                return taskResult;
            }
        }
        
        // 2. 全局约束校验
        for (GlobalConstraintValidator validator : globalValidators) {
            ValidationResult globalResult = validator.validate(result, context);
            if (!globalResult.isSuccess()) {
                return globalResult;
            }
        }
        
        return ValidationResult.success();
    }
}

@Component
public class ChangeMouldCountGlobalValidator implements GlobalConstraintValidator {
    
    @Override
    public ValidationResult validate(ScheduleResult result, ScheduleContext context) {
        // 统计各班次换模次数
        Map<String, Long> changeCountByShift = result.getTasks().stream()
            .filter(ScheduleTask::isChangeMould)
            .collect(Collectors.groupingBy(
                ScheduleTask::getShiftCode,
                Collectors.counting()
            ));
        
        // 校验每日总数
        long totalChange = changeCountByShift.values().stream()
            .mapToLong(Long::longValue)
            .sum();
        if (totalChange > 15) {
            return ValidationResult.fail("每日换模次数超过限制(15台)，实际: " + totalChange);
        }
        
        // 校验早班次数
        long morningCount = changeCountByShift.getOrDefault("MORNING", 0L);
        if (morningCount > 8) {
            return ValidationResult.fail("早班换模次数超过限制(8台)，实际: " + morningCount);
        }
        
        // 校验中班次数
        long afternoonCount = changeCountByShift.getOrDefault("AFTERNOON", 0L);
        if (afternoonCount > 7) {
            return ValidationResult.fail("中班换模次数超过限制(7台)，实际: " + afternoonCount);
        }
        
        return ValidationResult.success();
    }
    
    @Override
    public String getName() {
        return "换模次数全局校验器";
    }
```

## 7. SOLID原则应用

### 7.1 单一职责原则（SRP）

每个类只负责一个功能领域：

| 类 | 职责 |
|------|------|
| `ScheduleStrategy` | 定义排程算法接口 |
| `ContinueScheduleStrategy` | 实现续作排程逻辑 |
| `NewSkuScheduleStrategy` | 实现新增排程逻辑 |
| `MachineMatcher` | 负责机台匹配逻辑 |
| `PrioritySorter` | 负责优先级排序逻辑 |
| `CapacityCalculator` | 负责产能计算逻辑 |
| `ConstraintValidator` | 负责约束校验逻辑 |

**优点：**
- 代码清晰易懂
- 易于维护和测试
- 降低类之间的耦合

### 7.2 开闭原则（OCP）

对扩展开放，对修改关闭：

```java
// 新增一种排程策略，无需修改现有代码
@Component
public class SpecialPeriodScheduleStrategy implements ScheduleStrategy {
    
    @Override
    public ScheduleResult execute(ScheduleContext context) {
        // 实现特殊时期（如春节、国庆）的排程逻辑
    }
    
    @Override
    public boolean supports(ScheduleContext context) {
        return isSpecialPeriod(context.getScheduleDate());
    }
}
```

**应用场景：**
- 新增排程策略
- 新增约束校验器
- 新增机台过滤器
- 新增事件观察者

### 7.3 里氏替换原则（LSP）

子类可以替换父类而不影响程序正确性：

```java
// 所有策略实现类都可以替换 ScheduleStrategy 接口
ScheduleStrategy strategy = new ContinueScheduleStrategy();
ScheduleResult result = strategy.execute(context); // 正常工作

strategy = new NewSkuScheduleStrategy();
result = strategy.execute(context); // 同样正常工作
```

### 7.4 接口隔离原则（ISP）

客户端不依赖不需要的接口：

```java
// 细粒度的接口定义
```

### 7.5 依赖倒置原则（DIP）

依赖抽象而非具体实现：

```java
// 依赖抽象接口

    private Map<String, AbstractScheduleTemplate> scheduleTemplates;

            .collect(Collectors.toMap(

public class ScheduleStrategyContext {

    private List<ScheduleStrategy> strategies;
    
    public ScheduleResult schedule(ScheduleContext context) {
        for (ScheduleStrategy strategy : strategies) {
            if (strategy.supports(context)) {
                // 调用抽象接口方法
                return strategy.execute(context);
            }
        }
    }
}
```

```java
/**
 * 并行排程处理器
 * 对于相互独立的SKU，可以并行处理
 */
@Service
public class ParallelScheduleProcessor {
    
    @Autowired
    private ExecutorService executorService;
    
    public List<ScheduleTask> processParallel(List<SkuPlan> skus, ScheduleContext context) {
        // 按结构分组，同结构的SKU需要串行处理（共享资源）
        Map<String, List<SkuPlan>> groupByStructure = skus.stream()
            .collect(Collectors.groupingBy(SkuPlan::getStructureName));
        
        List<Future<List<ScheduleTask>>> futures = new ArrayList<>();
        
        for (List<SkuPlan> structureSkus : groupByStructure.values()) {
            Future<List<ScheduleTask>> future = executorService.submit(() -> {
                return processStructureSkus(structureSkus, context);
            });
            futures.add(future);
        }
        
        // 收集结果
        List<ScheduleTask> allTasks = new ArrayList<>();
        for (Future<List<ScheduleTask>> future : futures) {
            try {
                allTasks.addAll(future.get());
            } catch (Exception e) {
                log.error("并行处理失败", e);
            }
        }
        
        return allTasks;
    }
}
```

```java
/**

@Component
public class ScheduleCacheManager {
    
    @Autowired
    private StringRedisTemplate redisTemplate;
    
    private static final String CACHE_PREFIX = "schedule:";
    private static final long CACHE_TTL_HOURS = 24;
    
    /**
     * 缓存基础数据
     */
    public void cacheBaseData(String factoryCode, Date date, ScheduleBaseData data) {
        String key = buildKey(factoryCode, date, "base");
        redisTemplate.opsForValue().set(
            key, 
            JSON.toJSONString(data),
            CACHE_TTL_HOURS,
            TimeUnit.HOURS
        );
    }
    
    /**
     * 获取缓存的基础数据
     */
    public ScheduleBaseData getCachedBaseData(String factoryCode, Date date) {
        String key = buildKey(factoryCode, date, "base");
        String json = redisTemplate.opsForValue().get(key);
        if (json != null) {
            return JSON.parseObject(json, ScheduleBaseData.class);
        }
        return null;
    }
    
    /**
     * 清除缓存
     */
    public void clearCache(String factoryCode, Date date) {
        String pattern = CACHE_PREFIX + factoryCode + ":" + formatDate(date) + ":*";
        Set<String> keys = redisTemplate.keys(pattern);
        if (!CollectionUtils.isEmpty(keys)) {
            redisTemplate.delete(keys);
        }
    }
    
    private String buildKey(String factoryCode, Date date, String type) {
        return CACHE_PREFIX + factoryCode + ":" + formatDate(date) + ":" + type;
    }
    
    private String formatDate(Date date) {
        return new SimpleDateFormat("yyyyMMdd").format(date);
    }
}
```

```java
    private void saveScheduleDetails(List<ScheduleTask> tasks) {

            .collect(Collectors.toList());

            List<LhScheduleResult> batch = entities.subList(
                i, Math.min(i + batchSize, entities.size())
            );
            scheduleResultMapper.insertBatch(batch);
        }
```
