### 📋 硫化排程流程完整梳理

这套硫化排程流程分为 **6 大核心模块**，按顺序执行，各模块内部又包含细化子流程，整体逻辑从「数据准备」到「排产执行」再到「结果发布」，形成完整闭环。

---

### 🔍 模块总览（从开始到结束） 

```
开始 → S4.1 前置校验与数据清理 → S4.2 基础数据初始化 → S4.3 排程调整与SKU归集 → S4.4 续作规格排产 → S4.5 新增规格排产 → S4.6 结果校验与发布保存 → 结束
```


---

### 🧩 各模块详细拆解

#### 1. S4.1 前置校验与数据清理
**核心目标**：防止重复排程、清理历史脏数据，确保排程环境干净。
- 关键步骤：
  1. 选择排程日期（默认T+2日）
  2. 校验MES下发状态：
     - 已下发 → 禁止排程
     - 未下发 → 再校验是否正在排程
       - 正在排程 → 禁止重复排程
       - 未排程 → 删除旧排程数据 + 清理旧日志
- 结果：只有通过校验，才能进入下一步初始化。

---

#### 2. S4.2 基础数据初始化
**核心目标**：加载并校验排程所需的所有基础数据，保证数据完整性与一致性。
- 关键步骤：
  1. 初始化以下核心基础数据（对应 硫化排程实体类.txt文档 已创建好的实体类）
     - 月生产计划
     - 工作日历
     - SKU日硫化产能
     - 硫化参数
     - 设备停机计划
     - SKU与模具关系
     - 硫化机台信息
     - 模具清洗计划
     - 月底计划余量
     - 硫化排程各班次完成量
     - 物料信息
     - MES硫化在机信息
     - 硫化定点机台信息
     - 硫化机胶囊已使用次数
     - 设备保养（精度）计划
  2. 完整性校验：检查数据是否加载成功、关键字段是否为空
  3. 一致性校验：
     - 机台是否在启用列表中
     - SKU是否存在可用模具
  4. 封装标准化数据对象，生成初始化日志
- 结果：数据校验通过后，才能进入排程调整；若失败则流程终止。

---

#### 3. S4.3 排程调整与SKU归集
**核心目标**：基于前日排程修正产量，按结构归集SKU并标记「收尾/常规」「续作/新增」属性。
- 关键步骤：
  1. 复制前日排程到当前日期，修正实际完成量与计划量的差额
     - 欠产 → 追加到T日早班
     - 超产 → 冲抵T+1日计划
  2. 按结构归集SKU，计算硫化余量：
     - 硫化余量 = 月度计划量 - 已完成合格量
     - 若余量 ≤ 机台8班总产能 → 标记为「收尾SKU」
     - 否则 → 标记为「常规任务」
  3. 区分「续作SKU」（前日在产延续）和「新增SKU」（新排产）
- 结果：输出带属性标记的SKU归集结果，为后续排产提供依据。

---

#### 4. S4.4 续作规格排产
**核心目标**：对前日延续的SKU进行产能分配、胎胚库存匹配与降模处理。

- 关键步骤：
  1. 换活字块排产：匹配同胎胚/模具SKU，计算开产时间（避开清洗计划）
  2. 续作收尾判定：再次确认SKU是否为收尾状态，分配待排产量
  3. 班次计量分配：按「夜班→早班→中班」顺序分配产能，直到待排产量为0
  4. 胎胚库存调整：根据库存上限调整计划量（收尾SKU优先匹配库存）
  5. 降模排产：若产能不足，优先让胶囊使用次数少的机台下机，释放产能给高次数机台
- 结果：完成续作SKU的机台、班次、计划量分配。

---

#### 5. S4.5 新增规格排产
**核心目标**：对新SKU进行优先级排序、机台匹配、换模均衡、首检均衡与开产时间计算。
- **关键子流程**：
  - **S4.5.17 换模次数均衡**：控制早/中班换模次数（早≤8，中≤7，总≤15），均衡分配到换模少的班次，超限则标记未排产
  - **S4.5.首检均衡**：
    1. 收集待处理任务（换模/喷砂清洗/保养/维修，排除干冰清洗）
    2. 初始化早/中班操作数计数器（`morning_count=0`/`afternoon_count=0`）
    3. 遍历任务：优先分配到操作数更少的班次；若中班可安排（14:00-20:00 容纳换模+首检）则分配到中班，否则顺延到次日早班
    4. 更新对应班次计数器，保证班组负载均衡
  - **S4.5.18 开产时间计算**：结合保养/维修/清洗计划，避开模具交替重叠，计算准确开产时间
- 主流程步骤：
  1. 按发货锁定/生产延误等优先级排序新增SKU
  2. 筛选可用机台，匹配模具/胶囊，生成候选机台列表
  3. 优先匹配试制/小批量SKU到指定机台
  4. 换模次数均衡校验 → 超限则排产失败
  5. **首检均衡分配**：将需首检任务均衡分配到早/中班（或次日早班）
  6. 计算开产时间，分配计划量，扣减停机/清洗产能
  7. 标记未排产SKU的原因（无模具/换模超限/无机台）
- 结果：生成新增SKU的排产结果清单（已排+未排）。

---

#### 6. S4.6 结果校验与发布保存
**核心目标**：最终校验排程结果，生成模具交替计划（含首检均衡），保存并准备发布。
- **关键子流程（S4.6.2 生成模具交替计划）**：
  1. 准备换模机台列表，按「后规格可上机时间」排序
  2. 避开维修/保养计划，分配换模时间到早/中班
  3. 控制单日换模总次数（≤15），均衡早/中班次数
  4. **首检均衡分配**：对需首检的换模/清洗任务，按早/中班操作数均衡分配，遵守中班14:00-20:00时间窗口，不足则顺延
  5. 生成模具交替计划与日志
- 主流程步骤：
  1. 排程后置校验 → 不通过则终止
  2. 生成模具交替计划（含首检均衡）
  3. 生成批次号/工单号，设置为「未发布」状态
  4. 保存排程结果到数据库
- 结果：排程完成，等待后续MES下发。

---

### ✨ 核心逻辑亮点
1. **分层控制**：从前置校验→数据初始化/校验→排程调整→续作/新增排产→结果发布，层层递进，避免逻辑混乱。
2. **多维度约束均衡**：
   - 换模次数：早/中班换模次数上限控制
   - 首检任务：早/中班操作数负载均衡，严格遵守中班时间窗口（14:00-20:00）
   - 产能与库存：胎胚库存、机台产能约束下的计划量分配
3. **异常处理**：每个模块都有异常分支（如数据缺失、换模超限、首检任务无法安排），明确标记未排产原因，便于追溯。
4. **状态管理**：SKU属性标记（收尾/常规、续作/新增）贯穿全流程，指导不同排产策略。
5. **首检均衡价值**：平衡早/中班首检操作负载，避免单班组过载；严格遵守时间窗口，保证首检任务可执行，不影响正常生产节奏。

---

### 📌 流程调用关系
- S4.5 新增排产 → 调用 **S4.5.17（换模均衡）** → **S4.5.首检均衡** → **S4.5.18（开产时间计算）**
- S4.6 结果发布 → 调用 **S4.6.2（模具交替计划）**，其中包含**首检均衡**子流程
- 所有模块依赖 S4.1的前置数据准备与校验、S4.2 的基础数据初始化
